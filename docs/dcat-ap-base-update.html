1 · Before → after replaces partial base

The existing dcat-ap-profile template was a partial sketch and named its dataset class DcatDatasetProfile (a profile, not the canonical base) — so RQ2 extensions couldn’t cleanly inherit from it. The new base fixes both.

Before — dcat-ap-profile

8 classes · 16 properties

  • Partial: no DcatDataService, CatalogRecord, DatasetSeries, value classes
  • Dataset class named DcatDatasetProfile — not the base RQ2 extends
  • Obligations on global slots → title forced mandatory everywhere
  • No applicableLegislation, no controlled-vocab notes
→
After — dcat-ap-full-profile

29 classes · 70 properties

  • Canonical base classes DcatCatalog / Dataset / Distribution / DataService (+Series, Record, Resource)
  • 21 value/range classes (vCard, SKOS, ADMS, SPDX, ODRL, PROV, ELI…)
  • Obligations moved to per-class slot_usage — same slot can differ per class
  • Adds applicableLegislation, versioning terms, named controlled vocabularies

2 · The key change — obligations moved to slot_usage

Global slots now declare only reusable identity (URI · range · default cardinality). Each class refines requiredness / obligation / cardinality / usage notes locally.

# BEFORE — global slot carried obligation
slots:
  title:
    slot_uri: dcterms:title
    required: true            # forced on EVERY class
    annotations:
      requirement_level: mandatory
# AFTER — identity global, obligation per-class
slots:
  title: {slot_uri: dcterms:title, range: string, multivalued: true}

classes:
  DcatDistribution:
    slot_usage:
      title: {annotations: {requirement_level: optional}}
  DcatDataset:
    slot_usage:
      title: {required: true, annotations:{requirement_level: mandatory}}

3 · Same property, different obligation per class

Exactly what slot_usage unlocks — and what the new regression tests assert. Read across a row: one reused property, different rule per class.

property ↓ / class → CatalogDatasetDistributionDataServiceCatalogRecord
M mandatory (required, sh:minCount 1) R recommended O optional · not on this class

Notice title is M on Catalog/Dataset/DataService but O on Distribution/CatalogRecord; modified is O on Dataset but M on CatalogRecord — impossible to express with a single global slot.

5 · What we are not claiming yet work in progress

Per your instruction — the template’s own conformance_status annotation records these gaps.

⚠ Known gaps (tracked in the template, not hidden)

  • Not conformance-grade. Title downgraded to “DCAT-AP 3.0 Base (WIP)”; full SEMIC conformance suite not yet run.
  • Controlled-vocabulary value bindings (sh:in over EU Vocabularies) are named, not yet enforced.
  • Editor exporter gap: the bundled general_ontology_editor SHACL/JSON-Schema generator reads obligations from global slots only — it does not yet consume slot_usage. Per-class minCounts are authoritative in the template (and tested) but won’t appear in that exporter’s output until it’s updated.
