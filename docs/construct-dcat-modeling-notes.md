# Construct-DCAT Modeling Notes

The profile editor focuses on discovery-level semantic interoperability. A dataset does not need to harmonize its internal model before it can be discovered, onboarded, or validated.

Construct-DCAT profile terms should describe dataset-level anchors such as:

- lifecycle phase;
- asset type;
- representation type;
- source system;
- ontology or controlled vocabulary;
- SKOS concept;
- AAS submodel;
- IFC entity;
- BOT/building element.

Use base terms for reused DCAT/DCAT-AP vocabulary, profile terms for constraints over reused vocabulary, and extension terms for new `cx:` vocabulary introduced by the Construct-DCAT profile.
